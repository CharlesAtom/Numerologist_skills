# Numerologist Skills

AI 术数工程化仓库。目标是把传统术数知识转化为可维护、可追溯、可迭代的 `Skill + References` 体系，降低“大模型玄学幻觉”，提升推断稳定性。

## 项目目标

1. 让术数分析有明确流程：先确认口径，再排盘，再解读。
2. 让知识可维护：长文拆分成结构化 reference，而不是堆在对话上下文里。
3. 让结果可审计：每条结论都能追溯到规则来源与口径。

## 推荐协作方式（长篇经典接入）

当你要接入《渊海子平》或其他长篇文本时，建议按下面流程：

1. 分批输入：一次一个主题（如格局、六神、岁运）。
2. 结构化提炼：沉淀为“定义/条件/例外/禁忌/应期/自检”六段。
3. 形成独立 reference：例如 `references/yuanhai-ziping.md`。
4. 接入 `SKILL.md` 的 `Read` 列表与口径触发词。
5. 增加模板：在 `qa-templates.md` 增加该口径的开场与输出模板。

## 写作规范（建议）

1. `SKILL.md` 负责流程与行为约束，不塞过多原始术语。
2. `references/*.md` 负责知识规则，每个文件单一职责。
3. 有争议规则必须写明“口径来源”和“冲突处理”。
4. 必须保留风险边界，不输出宿命式或恐吓式表述。

## 下一步升级建议（Roadmap）

### Skill 层

1. 增加“口径路由器”：当用户提到特定师承或关键词时自动切换参考文件。
2. 增加“冲突并列输出”：同一命盘可并列两种口径差异与原因。
3. 增加“低信息模式”：时间不准时自动生成候选盘差异结论。

1. 新增 `yuanhai-ziping.md`（渊海子平结构化版）。
2. 为每个口径补“正反例样本库”（`references/cases-*.md`）。
3. 增加“术语映射表”：同义术语、古诀到现代语言的映射。
4. 增加“应期规则库”：大运、流年、交运、余气的统一模板。

## 愿景

可持续维护、可多人协作、可复用到不同模型与产品形态。
